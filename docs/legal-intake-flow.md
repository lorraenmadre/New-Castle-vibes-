# Legal Intake Flow

The Legal Intake Flow defines how New Castle receives and processes legal, family, trust, business, and document-heavy materials.

New Castle is not a replacement for licensed legal counsel. It is a preparation and organization system.

## Purpose

The intake flow should help a user move from document overwhelm into structured review.

It should capture:

- Document type
- Source
- People involved
- Entities involved
- Dates
- Deadlines
- Risks
- Missing information
- Related WISH WELL house
- Recommended next action
- Human review need

## Intake Stages

### 1. Upload or Reference

The user uploads or links a document.

The system records:

- File name
- Upload date
- Source
- User notes
- Related matter or project

### 2. Identify

The system classifies the document.

Possible types:

- Court document
- Contract
- Agreement
- Trust document
- Entity document
- Insurance policy
- Financial record
- Health or benefits record
- Property document
- Grant document
- Business plan
- Operating agreement
- Draft
- Correspondence

### 3. Extract

The system extracts useful facts:

- Names
- Dates
- Deadlines
- Case numbers
- Claim numbers
- Account numbers, when relevant
- Addresses
- Entities
- Obligations
- Required actions

### 4. Summarize

The system creates a plain-language summary.

Summary should include:

- What the document is
- Why it matters
- What it asks or requires
- What the user may need to do next
- What information appears missing

### 5. Route

The system assigns:

- Primary WISH WELL house
- Secondary WISH WELL house, if needed
- Storage location
- Monday.com task recommendation
- Human review recommendation

### 6. Decide

The user or reviewer decides whether to:

- Store only
- Create a task
- Create a timeline
- Draft a response
- Send to attorney or professional
- Add to case packet
- Archive

## Human Review Rule

Any document involving rights, deadlines, court orders, contracts, taxes, insurance coverage, custody, medical treatment, or financial obligations should be flagged for human review.

The system can help prepare and organize.

It should not pretend to be a court, attorney, doctor, therapist, tax advisor, or financial advisor. Apparently civilization still insists those roles require actual humans.

Each intake should produce:


## Outcome

The legal intake flow makes New Castle useful as a document triage shell.

The system should reduce overwhelm by turning each document into a summary, a route, and a next action.
